Daniel has made a list in which each protein in the original dataset is listed alongside the cluster for which it received the highest score in HHOMP's search. It also lists which subclusters a cluster contains. It is saved in "spring 2012 final paper/Clusters associated with TMBs in our dataset and PPI indices.xls"

 

I searched through this list for all double mentions of a cluster. I found that:

1A0S has cluster 73 (which includes 18.1.1 as a sublcluster) as top cluster, and 1AF6 has 18.1.1 as a top cluster.

3CSL and 3FHH both had 22.4.6 as top cluster

1QJP and 3DZM had cluster75 and cluster165, respectively, both of which contain 8.1.1 and nn.31.1

2POR and 3PRN both had cluster131 as top cluster. 1E54 has cluster28 which contains 16.2.1 and 16.2.3, the clusters that make up cluster131.

1QFG had 22.1.4, and 3EFM had cluster18 which contains 22.1.4.

1T16 and 3DWO both have 14.1.1 as a top result, and 3BS0 has cluster71, which contains 14.1.1.

 

I aligned Daniel's aligned structures (from "pymol/structures") of 1T16 and 3DWO. The inplug region aligned very well, as did the periplasmic loops; these are very good templates for those regions. However, the extracellular loops did not align well, and to some extent could not have, because some loops are of different length. The strands aligned pretty much perfectly. 3BS0 on 1T16 is the same story.

 

Examining 3EFM and 1QFG, periplasmic loops and inplugs again align very well. However, 3EFM is missing many extracellular loops from the structure, so I did not evaluate these. Again, strands aligned pretty much perfectly.

 

In 2POR and 2PRN, usual story, except that sometimes the strands are of different length, extending different distances towards the extracellular space. Where strand 12 of 3PRN has a turn, strand 12 of 2POR is still a β strand.

 

1QJP and 3DZM have strands of about the same tilt, but 3DZM has two long periplasmic loops that 1QJP does not appear to have. The β strands of 3DZM are longer than those of 1QJP, and the loops extend deeper into the extracellular space.

from __future__ import division

import csv

 

def avg(filename):

    with open(filename, 'r') as f:

        numbers = [abs(float(x[0])) for x in csv.reader(f)]

 

    return sum(numbers)/len(numbers)

from Bio import AlignIO

 

def z_diff_blist(target, alignment_path, z_diff_path,

                 alignment_format = 'clustal'):

    # Make a list of residue ids

    stored.resis = list()

    cmd.iterate(target + ' & n. CA',

                'stored.resis.append(resi)')

    assert len(stored.resis) == len(set(stored.resis)),\

           'duplicates in resi list'

   

    # Retrieve sequences as MSA object

    with open(alignment_path, 'r') as f:

        alignment = AlignIO.read(f, alignment_format)

    # Check that first is 1AF6 and the second is 1A0S

    maltoporin = str(alignment[0].seq)

    assert str(filter(lambda x: x!='-', maltoporin))[:5]=='VDFHG',\

           'first seq not maltoporin'

    scry = str(alignment[1].seq)

    assert str(filter(lambda x: x!='-', scry))[:5]=='SGFEF',\

           'second seq not sucrose porin'

 

    # Retrieve z_diffs as list of floats

    with open(z_diff_path, 'r') as f:

        z_diffs = [abs(float(i)) for i in f]

  

    blist = list()

    z_differator = iter(z_diffs)

    resiterator = iter(stored.resis)

 

    for malt_resn, scry_resn in zip(maltoporin, scry):

        if malt_resn != '-' and scry_resn != '-':

            blist.append((resiterator.next(), z_differator.next()))

        elif malt_resn == '-' and scry_resn != '-':

            # No maltoporin residue corresponding to this scry position

            resiterator.next()

        elif malt_resn != '-' and scry_resn == '-':

            # A maltoporin residue that doesn't map onto anywhere

            pass

 

    return blist

 

def color_b(target, blist):

    cmd.select('on_blist', 'none')

    for resi, b in blist:

        cmd.alter(target + ' & resi ' + resi,

                  'b = ' + str(b))

        cmd.select('on_blist', 'on_blist | resi ' + resi)

 

def color_by_z_diff(target, alignment_path, z_diff_path,

                    alignment_format = 'clustal'):

    # This is the function the user should actually type into the console

   

    blist = z_diff_blist(target, alignment_path, z_diff_path,

                         alignment_format = alignment_format)

    color_b(target, blist)

    return len(blist)

 

PyMOL commands to color a protein after b factors have been set. Blue corresponds to low abs(zdiff), red to high abs(zdiff). 7 Å abs(zdiff) or above is as red as it gets; by that point, the z prediction is completely wrong, and differentiating 7 from 40 would wash out all the differencews in the lower range. Intermediate range is colored white. Residues for which zdiff is not defined (residues on the template to which no residue in the sequence of interest was map) are colored black.

PyMOL>spectrum b, cyan_white_magenta, on_blist, 0, 7

PyMOL>color black, !on_blist

    cmd.spectrum('b',rainbow,'on_blist',0,7)
